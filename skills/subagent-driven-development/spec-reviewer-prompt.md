# 規格符合度審查提示範本

在派出規格符合度審查子代理時使用此範本。

**目的：**確認實作者只做了需求所要求的內容（不多不少）

```
Task tool (general-purpose):
  description: "審查任務 N 的規格符合度"
  prompt: |
    你正在審查實作是否符合其規格。

    ## 需求內容

    [FULL TEXT of task requirements]

    ## 實作者聲稱完成的內容

    [From implementer's report]

    ## CRITICAL: 不要相信回報

    實作者完成得異常迅速。他們的回報可能不完整、
    不準確或過度樂觀。你**必須**自行獨立驗證。

    **不要：**
    - 直接相信他們實作了什麼
    - 相信他們對完整性的宣稱
    - 接受他們對需求的詮釋

    **要做：**
    - 閱讀他們實際寫的程式碼
    - 逐行對照實作與需求
    - 檢查他們聲稱完成但實際缺漏的部分
    - 尋找他們沒提到的額外功能

    ## 你的工作

    閱讀實作程式碼並確認：

    **缺漏需求：**
    - 是否實作了所有需求？
    - 是否有被跳過或遺漏的需求？
    - 是否聲稱某功能可運作但其實沒有實作？

    **多做/不必要的工作：**
    - 是否做了未被要求的功能？
    - 是否過度工程或加入不必要的功能？
    - 是否加了規格外的「加值功能」？

    **誤解需求：**
    - 是否用不同於預期的方式解讀需求？
    - 是否解決了錯的問題？
    - 是否做對了功能但方法錯誤？

    **請以讀程式碼驗證，不要相信回報。**

    回報：
    - ✅ 符合規格（檢查程式碼後完全吻合）
    - ❌ 發現問題：[具體列出缺漏或多餘內容，並附檔案:行號]
```
